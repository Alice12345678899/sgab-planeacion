# Estimación del Proyecto

## Suposiciones de la estimación
- El proyecto será desarrollado por un equipo pequeño.
- No se cuenta con experiencia previa en sistemas académicos complejos.
- El alcance del sistema es limitado a funciones básicas.
- La estimación busca reducir incertidumbre, no exactitud matemática.

## Descomposición funcional
El sistema se divide en los siguientes módulos:
- Gestión de usuarios
- Gestión de alumnos
- Gestión de materias
- Gestión de calificaciones
- Reportes básicos

## Estimación de esfuerzo

| Módulo                     | Esfuerzo estimado (horas/persona) |
|---------------------------|-----------------------------------|
| Gestión de usuarios       | 25                                |
| Gestión de alumnos        | 30                                |
| Gestión de materias       | 20                                |
| Gestión de calificaciones | 35                                |
| Reportes básicos          | 20                                |
| **Total estimado**        | **130 horas**                     |

La estimación se realizó mediante descomposición y analogía con proyectos académicos similares.

## Nivel de incertidumbre
El nivel de incertidumbre es **medio**, debido a la posible ambigüedad en requisitos y a la falta de experiencia previa del equipo en sistemas académicos reales.

# Calidad del Software

La calidad del software se define como el grado en el que un sistema satisface necesidades explícitas e implícitas. En el proyecto SGAB, la calidad es fundamental para garantizar un sistema confiable y útil para el entorno académico.

## Usabilidad
La usabilidad es clave debido a que el sistema será utilizado por usuarios con distintos niveles de conocimiento tecnológico.  
Una interfaz clara y fácil de aprender reduce errores, tiempos de capacitación y resistencia al uso del sistema.

Si la usabilidad no se cumple, los usuarios pueden cometer errores frecuentes, abandonar el sistema o recurrir a procesos manuales.

## Seguridad
El sistema maneja información académica sensible, como calificaciones y datos personales.  
La seguridad es necesaria para proteger la confidencialidad e integridad de los datos, así como para evitar accesos no autorizados.

La falta de seguridad puede provocar pérdida de información, manipulación de datos y problemas legales o institucionales.

## Mantenibilidad
La mantenibilidad permite que el sistema pueda ser corregido o mejorado con el tiempo.  
Un sistema mantenible facilita la adaptación a cambios en reglas académicas y reduce el costo de evolución.

Si el sistema no es mantenible, cualquier cambio implicará retrabajo, mayor riesgo de errores y altos costos de modificación.
